### 一、 核心 Loss 函数公式化设计

为了实现“不求最大，只求成比例”的分布匹配，同时兼顾多样性与稳定性，我们将总损失函数定义为：

$\mathcal{L}_{total} = \mathcal{L}_{Soft-TB} + \lambda_1 \mathcal{L}_{Entropy} + \lambda_2 \mathcal{L}_{Ref}$

### 1. 核心对齐项：Softmax-Trajectory Balance (Soft-TB)

该项强制 27 条路径的概率分布与它们的奖励分布达成一致 ：

$\mathcal{L}_{Soft-TB} = \sum_{i=1}^{27} \left( \log \frac{P_{\theta}(\tau_i)}{\sum_{j=1}^{27} P_{\theta}(\tau_j)} - \log \frac{R(x_{0,i})}{\sum_{j=1}^{27} R(x_{0,j})} \right)^2$

- **物理意义**：通过组内比例匹配，绕过了对全局配分函数 $Z$ 的直接拟合，利用相对流量实现分布对齐。

### 2. 多样性保障项：Particle-based Entropy ($\mathcal{L}_{Entropy}$)

为了防止 27 个分支在奖励诱导下向同一个局部最优解坍缩，引入基于 RBF 核函数的斥力项：

$\mathcal{L}_{Entropy} = \frac{1}{K(K-1)} \sum_{i=1}^{K} \sum_{j \neq i}^{K} \exp \left( -\frac{\| \phi(x_{0,i}) - \phi(x_{0,j}) \|^2}{h} \right)$

- **实现细节**：$\phi(x)$ 是图像在 Flux 潜空间或 CLIP 空间的特征向量，$h$ 是带宽参数。
- **物理意义**：当两个分支生成的图像在语义上太接近时，损失函数会增大，产生“斥力”迫使模型探索奖励景观中的不同波峰（Modes）。

### 3. 基础能力维持项：Reference Constraint ($\mathcal{L}_{Ref}$)

参考 **FlowRL** 的做法，利用预训练模型作为约束，防止模型在强化学习过程中“跑偏” ：

$\mathcal{L}_{Ref} = \sum_{i=1}^{27} \left( \frac{1}{|\tau|} \log \pi_{\theta}(\tau_i) - \frac{1}{| \tau |} \log \pi_{ref}(\tau_i) \right)^2$

- 
    
    **实现细节**：引入长度归一化（$1/|\tau|$）以平衡不同复杂程度路径的梯度贡献 。
    
- **物理意义**：它像一条“安全带”，确保模型在追求高奖励分布的同时，不丢失原始生成的图像质量和概率流流形。

### 二、 Z-AFT 项目全维度方案详解

### 1. 阶段一：$Z_{\phi}$ 难度感知预热 (Warm-up Phase)

系统首先训练一个轻量级的 MLP 模块 $Z_{\phi}(c)$。

- 
    
    **逻辑**：$Z$ 预测当前 Prompt 在原始模型下的“奖励总积分” 。
    
- **用途**：$Z$ 不参与 Soft-TB 的梯度回传，而是作为**算力调度器**。
    - **低 $Z$ (难题)**：模型感知的对齐难度大，触发“深度思考”模式，提前分支点 $t_{split}$ 并增加采样步数。
    - **高 $Z$ (易题)**：模型感知自信度高，直接走 ODE 或晚期分叉，节省算力。

### 2. 阶段二：3-Step 27-Branch 树状采样 (Adaptive Tree Sampling)

在扩散过程中，我们不采用独立的随机采样，而是构建一个三层树状结构：

- **拓扑结构**：
    - 在 $t_1, t_2, t_3$ 三个关键时间点（分支点）进行 SDE 加噪分叉。
    - 每个分支点分出 3 个子节点，最终形成 $3 \times 3 \times 3 = 27$ 个终端样本。
- **概率累积**：
每条完整路径的对数概率 $\log P(\tau)$ 是这三个加噪步进概率的乘积：$\log P(\tau) = \sum_{k=1}^3 \log P_{SDE}(x_{k} | x_{k-1})$。这不仅大幅降低了从 $t=1$ 全量计算的显存压力，还利用了局部轨迹平衡（Sub-TB）的优势 。

### 3. 阶段三：分布匹配优化 (Flow-Balanced Optimization)

利用上述设计的 $\mathcal{L}_{total}$ 进行反向传播。

- **核心理念**：我们的目标不是为了得到一张“最高分”的图（像 GRPO 那样追求 $R$ 的峰值） ，而是要让模型学会：**如果一个 Prompt 有三种合理的画法，那么模型采样出这三种画法的概率应该严格对应它们的奖励权重** 。
- **参数更新**：通过优化 $\theta$，调整扩散模型的速度场 $v_{\theta}$，使得从起点出发的概率流能够根据奖励地形（Reward Landscape）进行精准的“流量分配”。

### 一、 核心创新：引入重要性采样 (IS)

在扩散模型中引入 IS 的核心目的是解决“采样不一致”问题：即当模型参数 $\theta$ 更新后，之前采样得到的轨迹 $\tau$ 就不再属于当前策略的分布 。

### 1. 重要性权重定义

仿照 FlowRL 的设计 ，我们定义轨迹级的重要性权重 $w$：

+1

$w = \text{clip}\left( \frac{\pi_{\theta}(\tau)}{\pi_{old}(\tau)}, 1-\epsilon, 1+\epsilon \right)^{\text{detach}}$

- 
    
    **实现逻辑**：为了防止策略漂移过大，计算权重时对当前策略 $\pi_{\theta}$ 进行 `detach` 处理 。
    
- 
    
    **多次更新**：有了 $w$ 的修正，您可以对同一批 27 分支样本进行 4-8 次梯度迭代，极大地提升了 Flux 等重型模型的训练效率 。
    

### 二、 关于 $Z$ 的定义：从“学习”转向“在线估计”

您提出直接用**组内样本奖励的均值**来估计 $Z$。

### 1. 理论可行性

- 
    
    **参考证据**：FlowRL 提到，**Kimi-K1.5** 确实使用了采样奖励的经验均值（Empirical Mean）来近似归一化常数 $Z$ 。
    
- **您的优势**：在 **Soft-TB** 框架下，您的梯度更新已经不再依赖绝对的 $Z$（因为它在比例中被抵消了）。因此，将 $Z$ 定义为“难度”，并用组内均值 $\bar{R}$ 来在线估计是完全合理的。
- **调度逻辑**：如果 $\bar{R}$（在线估计的难度）持续较低，则说明当前提示词属于难题，系统自动触发“早期分叉”或“增加步长”的 Tree 策略。

### 三、 完整项目方案：Z-AFT (Z-Adaptive Flow-Tree)

我们将您的想法整合成一套严谨的学术方案：

### 1. 系统架构：三阶 27 分支树 (3-Step 27-Branch Tree)

- **结构**：固定在扩散过程的三个关键时刻（例如 $t=0.8, 0.5, 0.2$）进行分叉。
- **分叉**：每个点分出 3 个 SDE 噪声分支，最终产生 $3^3=27$ 条路径。
- **概率**：路径对数概率为三个关键加噪步的乘积：$\log P(\tau) = \sum_{step=1}^3 \log P_{SDE}$。

### 2. 最终 Loss 函数设计

$\mathcal{L}_{total} = w \cdot \mathcal{L}_{Soft-TB} + \lambda_1 \mathcal{L}_{Entropy} + \lambda_2 \mathcal{L}_{Ref}$

- **分布对齐 (Soft-TB)**：实现路径概率与奖励比例的匹配。
    
    $\mathcal{L}_{Soft-TB} = \sum_{i=1}^{27} \left( \log \frac{P_{\theta}(\tau_i)}{\sum_{j=1}^{27} P_{\theta}(\tau_j)} - \log \frac{R(x_{0,i})}{\sum_{j=1}^{27} R(x_{0,j})} \right)^2$
    
- **多样性斥力 (Entropy)**：利用 RBF 核函数防止 27 个分支语义趋同。
    
    $\mathcal{L}_{Entropy} = \text{mean}_{i,j} \left[ \exp \left( -\frac{\| \phi_i - \phi_j \|^2}{h} \right) \right]$
    
- 
    
    **稳定性约束 (Reference)**：防止模型偏离原始分布 。
    
    +1
    
    $\mathcal{L}_{Ref} = \sum_{i=1}^{27} \left( \frac{1}{|\tau|} \log \pi_{\theta} - \frac{1}{|\tau|} \log \pi_{ref} \right)^2$
